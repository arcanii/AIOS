## Priority 1: Signal Phase 2 -- Handler Invocation

Current state: signals are registered (sigaction) and delivered
(kill sets pending bit), but handler functions are not yet invoked.

Next steps:
- Check sig_pending on syscall return paths in aios_posix.c
- For pending unblocked signals with SA_SIGINFO or handler != SIG_DFL/SIG_IGN:
  invoke handler directly (cooperative model)
- SIG_DFL behavior: SIGTERM/SIGINT -> _exit(), SIGCHLD -> ignore,
  SIGPIPE -> _exit(), etc.
- SIGCHLD delivery to parent on child exit (pipe_server PIPE_EXIT path)

## Priority 2: Pipeline Error Recovery

run_pipeline has no cleanup on mid-pipeline fork failure. If fork fails
for segment N of M, earlier children block forever.

Fix: PIPE_KILL to already-forked children, close all pipe fds,
waitpid for killed children.

## Priority 3: Multi-Chain Support (&&/|| recursion)

"true && echo a && echo b" drops second chain. Need iterative
or recursive chain evaluation.

## Priority 4: POSIX Audit

Systematic audit of POSIX compliance:
- Create test program that reports implemented vs missing syscalls
- Test edge cases: dup2, symlink, chmod, chown, pipe()
- Fix access() for /etc/hostname (25/26 -> 26/26)

## Priority 5: Getty Robustness

- Fork failure retry with degraded fallback
- WIFSIGNALED logging on child crash
- Ctrl-C in login input

## Priority 6: PIPE_EXEC_WAIT Deadlock Recovery

Child crash between PIPE_FORK and PIPE_EXEC leaves parent blocked.
handle_child_fault should call check_exec_wait(child_idx).

## Priority 7: Static Buffer Cleanup

do_exec/do_exec_bg use static buffers for argv. Move to stack-local.
