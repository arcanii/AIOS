# AIOS Next Steps -- 2026-04-29a

Continuing from **v0.4.99**. Stability + logging infrastructure (Option B).

## Session Result: /var/log Mount + Log Drive Fix + LOG_* Migration

### Goal
Migrate file logging from `/log/aios.log` to `/var/log/aios.log` (FHS-style),
fix a long-standing dual virtio-blk bug, and start migrating ad-hoc printfs in
servers to the structured `AIOS_LOG_*` macro infrastructure.

### Changes

**1. /var/log mount point (P1)**
- `disk/rootfs/var/log/.keep` -- new mount point directory in rootfs overlay
- `boot_fs_init.c`: log drive now mounts at `/var/log` instead of `/log`
- `aios_log.c`: messages updated, file path documentation
- `procfs.c`: `/proc/mounts` shows `/var/log`, `vfs_stat("/var/log")` check
- `fs_server.c`, `aios_log.h`: comments updated

**2. Dual virtio-blk warmup fix (P1.5, bonus)**
- Long-standing bug: with both system disk and log drive present, ext2 reads
  on the system disk silently fail intermittently after `plat_blk_init_log`
  configures the second virtio-blk device.
- Symptom: `/etc/network.conf`, `/etc/environment`, `/etc/passwd`, and
  `/bin/aios/getty` all return `vfs_read = -1`, even though the disk image
  is correct.
- Workaround: do a single dummy read of system disk sector 2 immediately
  after `plat_blk_init_log()` returns. This "warmup" reliably clears the
  bad state. Root cause unconfirmed -- suspected stale IRQ status or
  used-ring race between the two virtio-blk devices in QEMU.
- Implementation: `boot_fs_init.c` calls `plat_blk_read(2, warmup_buf)`
  after log drive setup but before `ext2_init` on the log drive.

**3. Structured logging migration (P2, partial)**
- 12 informational/error printfs migrated to `LOG_*` macros across 3 servers
- `exec_server.c`: ELF load info, fork failures, ELF errors -> LOG_INFO/ERROR
- `thread_server.c`: thread create/join messages -> LOG_INFO/ERROR
- `pipe_server.c`: signal delivery, ref counting -> LOG_DEBUG/ERROR
- All log entries now have:
  - Timestamp `[HH:MM:SS]`
  - Level tag `INF/WRN/ERR/DBG`
  - Module name (`exec`, `thread`, `pipe`)
  - Persistent storage in `/var/log/aios.log`
  - Real-time serial echo
  - `/proc/log` ring buffer access for dmesg-style queries
