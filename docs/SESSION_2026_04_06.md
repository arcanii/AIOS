## Key Learnings

### 1. reap_check() in fs_thread causes premature child death
Problem: reap_check() polled fault EPs inside fs_thread_fn after every
IPC message. When a fork+exec child called open(), fs_thread processed the
FS_STAT request, then reap_check() consumed the child fault EP before the
child finished, killing it mid-execution.

Fix: Removed reap_check() from fs_thread entirely. The PIPE_EXIT +
PIPE_WAIT flow in pipe_server handles the complete lifecycle:
child exit_cb -> PIPE_EXIT -> fault -> waitpid via PIPE_WAIT -> zombie table.

Lesson: Never poll fault EPs from a server that children depend on for I/O.

### 2. PIPE_EXEC needs fresh fault_ep
Problem: After sel4utils_destroy_process, the old fault_ep retains
residual fault state. On SMP, the fault can arrive after seL4_NBRecv.

Fix: Allocate a fresh fault_ep for the exec-ed process.

### 3. CPIO-loaded processes cannot fork
Problem: process_config_elf() loads from CPIO but the VSpace does not
track page capabilities. vspace_get_cap() returns seL4_CapNull,
so fork_copy_region() fails.

Fix: Spawn mini_shell from disk via exec_thread (EXEC_RUN_BG).
Any process that needs fork() must be loaded from disk, not CPIO.

### 4. sbase tools need unbuffered stdout
Problem: musl defaults to full buffering when isatty() fails.
The AIOS exit path (VM fault) kills the process before musl flushes.

Fix: setvbuf(stdout, NULL, _IONBF, 0) in __wrap_main, plus
fflush(stdout) in aios_exit_cb before PIPE_EXIT.

### 5. __real_main needs weak attribute
Problem: mini_shell links aios_posix without --wrap=main.
__wrap_main calls __real_main which is undefined without the flag.

Fix: Declare __real_main with __attribute__((weak)).

### 6. Build: ninja + rm -rf loses .a files
Problem: EXCLUDE_FROM_ALL libraries are not rebuilt by default.
aios-cc linker fails with missing .a files.

Fix: After ninja, force: ninja sel4muslcsys sel4utils sel4simple
sel4simple-default sel4allocman sel4platsupport sel4vspace aios_posix
