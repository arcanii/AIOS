# AIOS Next Steps -- 2026-04-11f

Starting from **v0.4.79**, building to **v0.4.80**.

## Completed in v0.4.80

### Dynamic configuration system

- New shared key=value config parser (`config.h`, `config_parser.c`)
  - `cfg_parse_kv()`: line-by-line parser, skips `#` comments, static buffers
  - `cfg_get()`: key lookup in parsed config
  - `cfg_parse_ip()`: dotted-quad string to uint8_t[4]
  - Global config instances with hardcoded defaults as fallback
- Boot-time config loader (`boot_config.c`)
  - `cfg_load_hostname()`: reads /etc/hostname, strips newline, fallback "aios"
  - `cfg_load_network()`: reads /etc/network.conf, parses ip/gateway/mask keys
  - `cfg_load_environment()`: reads /etc/environment, parses KEY=value entries
  - `boot_load_config()`: master loader called before server threads start
- Boot log output: hostname, network config, env variable count

### Runtime network configuration

- `net_cfg_ip[4]`, `net_cfg_gw[4]`, `net_cfg_mask[4]` arrays replace hardcoded NET_IP_A/B/C/D macros
- net_stack.c: all 15 NET_IP/NET_GW references replaced with runtime arrays
- net_tcp.c: IP source address from runtime config
- boot_config.c copies parsed values to runtime arrays before net_server starts

### Dynamic environment variables

- `aios_posix.c`: static `aios_envp[]` replaced with mutable `env_storage[16][128]`
- `env_init_defaults()` in aios_init: reads /etc/environment via fetch_file IPC
- Reads /etc/hostname for HOSTNAME= if not in environment file
- Fills missing essentials with defaults (HOME, PATH, USER, SHELL, TERM, HOSTNAME)
- SHELL=/bin/dash from config (was hardcoded /bin/sh)

### Quick fixes

- auth_server: default shell /bin/sh -> /bin/dash (4 locations in aios_auth.c)
- uname fallback release: "0.4.37" -> "0.4.80" (posix_misc.c)
- /proc/mounts: shows /dev/vdb /log when log drive mounted (was hardcoded 2 entries)
- USER= env var set from login uid in __wrap_main (was always "root")

### Test infrastructure

- Test scripts moved to /bin/tests/ (avoids collision with sbase test binary)
- test_080.sh: 14 tests for config files, environment, hostname, mounts, pipes, ext2
