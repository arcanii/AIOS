## Priority 1: && / || with Pipes

Current: && chain detection runs before pipe detection.
`echo test1 | cat && echo test2 | cat` runs left side as "echo test1 | cat"
which works, but right side as literal "echo test2 | cat" without pipe parsing.
Fix: after && split, re-parse each side for pipes.

## Priority 2: Getty/Shell Separation

* root spawns getty (CPIO, 5 caps)
* getty: banner + login loop, EXEC_RUN mini_shell
* mini_shell: pure command interpreter (no login code)
* exit/logout returns to getty

## Priority 3: Remove exec_ep from mini_shell

exec_ep is no longer used for redirect/pipe paths (all fork+exec now).
Still used for: do_exec (regular commands), do_exec_bg (background).
Convert these to fork+exec. mini_shell becomes a pure PosixApp.

## Priority 4: Signal Infrastructure

* SIGCHLD, SIGINT, SIGTERM
* signal()/sigaction() in aios_posix

## Priority 5: Shared-Memory Pipes (v0.5.x)

Current IPC pipes: pipe_server round-trip per chunk (~900 bytes max).
Shared-memory: ring buffer with atomic head/tail. Zero IPC for data transfer.
