# AIOS Next Steps -- 2026-04-07g

## Starting from v0.4.57

## Completed in v0.4.57

* pthread --wrap linkage for CMake-built apps: AiosPosixApp macro
  only had --wrap=main; musl pthread_create ran instead of IPC
  wrapper, silently failing with EAGAIN. Added all 8 --wrap=pthread_*
  flags to AiosPosixApp and getty. Root cause of 5/79 posix_verify
  SKIPs.
* Thread return value propagation: thread_server reads x0 via
  seL4_TCB_ReadRegisters on faulted thread, sends retval in MR1.
  __wrap_pthread_join reads MR1 into *retval. Verified via
  posix_verify thread_retval=99 test.
* Cap revoke on thread cleanup: seL4_CNode_Revoke before
  vka_free_object in thread join path. Allows split allocator
  to reclaim untyped memory for thread reuse. Fixed second
  pthread_create failing with Insufficient memory.
* Allocator pool increase: 800 -> 1500 pages in aios_root.c
  ALLOCATOR_STATIC_POOL_SIZE. Provides headroom for concurrent
  thread creation across multiple processes.
* aios_get_thread_ep() getter added to aios_posix.c/h for
  diagnostic and future programmatic use.
* posix_verify V2: 79/79 PASS, 0 FAIL, 0 SKIP.
* chdir validation: aios_sys_chdir now calls fetch_stat before
  updating CWD. Rejects non-existent paths (-ENOENT) and
  non-directories (-ENOTDIR). mini_shell cd builtin checks
  chdir() return and prints error on failure instead of
  silently corrupting the prompt.
