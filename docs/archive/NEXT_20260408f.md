* **FS_PREAD (positioned read).** New IPC op (label 13) enables reading
  files at arbitrary offsets. Files larger than 4KB now readable via
  multiple IPC round-trips (~950 bytes each). The openat path uses
  fetch_stat for file size and on-demand reads via fetch_pread for
  large files. Required for tcc to read tccdefs.h (13KB).
  Files: ext2.c, vfs.c, vfs.h, ext2.h, fs_server.c, aios_posix.c,
  posix_file.c, posix_internal.h

* **FS_PWRITE (positioned write).** New IPC op (label 20) enables
  writing data at arbitrary offsets in existing files. Allocates new
  ext2 blocks as needed (direct + single indirect = up to 268KB).
  ext2_set_block_num allocates indirect blocks on demand.
  Files: ext2.c, vfs.c, vfs.h, ext2.h, fs_server.c, aios_posix.c,
  posix_internal.h

* **writev for regular files.** musl stdio uses writev() not write()
  for buffered I/O. The writev handler was missing the regular file
  path -- only handled pipes and devnull. Added FS_PWRITE-based
  writev for regular files. This was the critical fix enabling tcc
  to write output files.
  Files: posix_file.c

* **File write path rewritten.** Regular file write() now uses
  FS_PWRITE directly instead of the old FS_WRITE_FILE (which replaced
  the entire file each call, capped at 512 bytes). Writes are chunked
  at 800 bytes per IPC. Position tracked in fd. Supports lseek to
  arbitrary positions.
  Files: posix_file.c

* **ext2 builder duplicate directory fix.** install_rootfs now checks
  for existing directories (find_dir_entry) before creating new ones.
  Prevents duplicate /tmp entries when rootfs overlay contains
  directories already created by build().
  Files: scripts/ext2/builder.py

## Lessons Learned in v0.4.70

* **musl stdio uses writev, not write.** fwrite/fprintf/fclose go
  through musl buffered I/O which calls writev(). The POSIX shim
  writev handler only supported pipes and devnull, silently returning
  EBADF for regular files. This caused all stdio-based file output
  to produce 0-byte files. Fix: add FS_PWRITE path to writev.

* **File I/O was read-entire-on-open.** The original design loaded
  the entire file into a 4KB buffer at open() time. All reads came
  from that buffer. Files >4KB were truncated. The fix (FS_PREAD)
  keeps the old fast path for small files and uses on-demand IPC
  reads for large files.

* **File writes replaced entire file.** FS_WRITE_FILE called
  vfs_create which unlinked the old file and created a new one.
  Each write() call replaced the file. FS_PWRITE fixes this by
  writing at specific offsets without replacing.

* **malloc fails in AIOS child processes.** The morecore pool (4MB)
  is available per-process but malloc(64KB) returned NULL. Client-side
  write buffers via malloc are unreliable. Direct IPC (FS_PWRITE per
  write call) is more reliable and simpler.

* **tcc linking needs full AIOS runtime.** tcc -c works (preprocessor
  + codegen). tcc -o fails because the built-in linker cannot find
  AIOS runtime symbols (__sel4_start_c from sel4runtime). Next step:
  bundle all AIOS runtime libs into a single libaios.a and create a
  tcc-compatible CRT without --wrap dependency.
