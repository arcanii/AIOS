## Priority 1: ext2 Directory Cache (unlink verification)

ext2_unlink writes the updated directory block, but ext2_lookup
may still find the entry if the unlinked block is not in the
same cache slot. Current cache fixes CREATE path; verify UNLINK
path also works via cache. Add regression test to posix_verify.
Files: src/ext2.c

## Priority 2: Getty Robustness

* Fork failure retry with backoff (3 attempts, 500ms sleep)
* On persistent failure, fall back to mini_shell in-process
* WIFSIGNALED logging on child crash
* Ctrl-C (0x03) handling in login input and read_password()
  Files: src/apps/getty.c

## Priority 3: Signal Phase 3 -- Cross-Process Delivery

Current aios_sig_check() only runs on kill/tgkill return paths.
For cross-process signal delivery, the child needs to check
sig_pending on other syscall return paths (read, write, nanosleep).
Files: src/lib/posix_file.c, src/lib/posix_time.c

## Priority 4: Pipeline Cleanup on Mid-Fork Failure

If fork fails for segment N of M, earlier children are already running.
Fix: SIGKILL+reap already-forked children, close all pipe fds.
Files: src/apps/mini_shell.c

## Priority 5: getconf (last sbase tool)

Write scripts/gen_getconf.py to produce minimal getconf.h.
Would bring sbase to 99/99.
Files: scripts/gen_getconf.py

## Priority 6: Shared-memory pipes

Replace IPC round-trip with vka_alloc_frame mapped into both VSpaces.
Files: src/servers/pipe_server.c

## Priority 7: Allocator Memory Audit

The bump from 800 to 1500 pages is a band-aid. Audit untyped
consumption across boot, exec, fork, and thread creation to
right-size the pool. Consider whether vka_free_object properly
returns memory to the split allocator after CNode_Revoke, or
if a watermark approach is needed.
