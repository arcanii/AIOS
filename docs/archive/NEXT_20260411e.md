# AIOS Next Steps -- 2026-04-11e

Starting from **v0.4.78**, building to **v0.4.79**.

## Completed in v0.4.79

### O_NONBLOCK for pipes

- Added `is_nonblock` field to `aios_fd_t` struct
- `pipe2()` now honors O_NONBLOCK flag (0x800), sets both read and write ends
- `fcntl()` F_GETFL returns real flags (O_NONBLOCK, O_APPEND, O_RDWR)
- `fcntl()` F_SETFL toggles O_NONBLOCK on aios fds
- Server-side: PIPE_READ and PIPE_READ_SHM check MR2 nonblock flag
- Nonblocking read on empty pipe returns -1 (client maps to EAGAIN)
- All PIPE_READ callers updated to send 3 MRs (MR2=flags)

### futex stub (NR 98)

- FUTEX_WAKE (op 1): returns 0 (no waiters)
- FUTEX_WAIT (op 0): compares *uaddr with val, returns -EAGAIN if different, -ETIMEDOUT if equal
- Masks FUTEX_PRIVATE_FLAG (0x80) for compatibility
- Required by musl pthreads internal locking

### /proc/self/fd support (partial)

- `fstatat()` client-side intercept: /proc/self -> S_IFDIR, /proc/self/fd -> S_IFDIR, /proc/self/fd/N -> S_IFLNK
- `readlinkat()` extended: /proc/self/fd/N returns fd path (/dev/tty for 0-2, pipe:[0] for pipes, file path for aios fds)
- Server-side `procfs_stat()` returns S_IFDIR for /self, /self/fd, pid directories
- "self" added to /proc directory listing (12 entries, was 11)
- Synthetic getdents64 generator (`procfd_gen_dirents`) in both open and openat
- Known limitation: external command opendir does not route through client-side intercept (syscall dispatch investigation needed)

### close() pipe EOF fix

- `close()` now sends PIPE_CLOSE_WRITE when closing the last write reference to a pipe
- Checks: no other fd in table has same pipe_id as write end, stdout_pipe_id != pipe_id
- Prevents premature EOF in fork+dup2+exec (stdout still holds pipe)
- Fixes builtin-to-external pipelines: `echo hello | cat` now delivers EOF correctly
- Previously: close() never sent PIPE_CLOSE_WRITE, relying solely on process exit

### Stability improvements

- **Zombie overflow guard**: evicts slot 0 with warning when zombie table full (was silent data loss)
- **Fork error cleanup**: ReadRegisters failure now destroys child process and frees fault EP cap (was leaking)
- **reap_check() NOT integrated**: seL4_NBRecv on minted fault EPs steals messages from shared pipe_ep queue. Existing badge-based fault detection in main loop is correct and sufficient.
