## Key Bugs Found and Fixed

### 1. pipe2() fd allocation (THE root cause of 20+ failed attempts)

Both pipe fds got the same index because aios_fd_alloc() did not mark
the slot active before the second call. Single-line fix resolved everything.

### 2. close() sending PIPE_CLOSE destroyed pipe buffer prematurely

Reader never saw data because close(write_fd) destroyed the server-side
buffer. Fix: close() sends no server IPC at all.

### 3. close(write_fd) sending PIPE_CLOSE_WRITE before exec

In dup2+close+exec pattern, PIPE_CLOSE_WRITE fired before the program
even loaded. Fix: only aios_exit_cb signals EOF on process death.

### 4. SMP fork startup race

Parent forks two children rapidly; both call PIPE_EXEC simultaneously,
corrupting shared elf_buf. Fix: 20x seL4_Yield() between forks.

### 5. MAX_ACTIVE_PROCS exhaustion

8 slots minus 7 servers = 1 slot. Pipelines need 2-3 simultaneous.
Fix: increased to 16.
