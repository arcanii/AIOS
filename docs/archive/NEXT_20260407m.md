# AIOS Development -- NEXT_20260407m

## Starting from v0.4.63

## Priority 1: Allocator Memory Audit

The 1500-page untyped pool is the primary bottleneck. sbase_test
exhausts at 18 fork/exec cycles. Each fork+exec allocates a child
VSpace (page directory, page tables, TCB, CNode, endpoints).
Resources are not fully reclaimed after waitpid.

Tasks:
- Instrument vka_alloc calls: log untyped consumption per boot,
  per fork, per exec, per thread_create
- Identify leaked objects after waitpid (CNode slots, page frames,
  TCBs not revoked)
- Implement proper cleanup in exec_server reaper: seL4_CNode_Revoke
  on child CNode, vka_free_object for all child allocations
- Right-size the pool based on measured per-process cost
- Target: 40+ fork/exec cycles before exhaustion

Files: src/aios_root.c (allocator init), src/servers/exec_server.c
(reaper), src/servers/pipe_server.c (fork/wait lifecycle)

## Priority 2: ext2 Directory Cache Invalidation

touch/open(O_CREAT) creates files via FS_WRITE_FILE IPC, but
the fs_server directory cache still holds stale entries. fstatat
after file creation returns ENOENT. ls shows stale listings.

Tasks:
- Add cache_invalidate(parent_dir) call in fs_server after
  FS_WRITE_FILE and FS_MKDIR operations
- Verify with: touch /tmp/test && ls /tmp (should show file)
- Verify with: sbase_test touch test (currently skipped)

Files: src/servers/fs_server.c, src/ext2.c (cache layer)

## Priority 3: Shared-Memory Pipes

Replace IPC round-trip pipe model with vka_alloc_frame mapped
into both parent and child VSpaces. Current model copies data
through MRs (900 bytes max per IPC), which is slow and wastes
MR bandwidth.

Tasks:
- Allocate shared 4KB frame per pipe
- Map into writer and reader VSpaces
- Ring buffer protocol: write_head, read_tail, data[4088]
- Keep IPC for signaling (notify on write, block on empty)
- Measure: pipe throughput before/after

Files: src/servers/pipe_server.c, src/lib/posix_file.c (read/write)

## Priority 4: Shell Append Redirect (>>)

echo bbb >> file overwrites instead of appending. The shell
opens with O_WRONLY|O_CREAT|O_TRUNC instead of O_WRONLY|O_CREAT|O_APPEND.
Our posix shim openat ignores O_APPEND.

Tasks:
- Honor O_APPEND flag in openat: set f->pos = f->size after open
- Implement append-mode write in FS_WRITE_FILE (offset = file_size)
- Shell: ensure >> passes O_APPEND and does NOT pass O_TRUNC

Files: src/lib/posix_file.c, src/apps/mini_shell.c

## Priority 5: sbase_test Expansion

Once allocator and cache are fixed, expand sbase_test to cover
all 99 sbase tools. Current V4 covers 28 tests with 18 fork/exec.
Target: 50+ tests with 40+ fork budget.

Additional tools to test: sed, find, tr, cut, paste, join,
uniq, comm, diff (when available), dd, od, split, tee, xargs.

Files: src/apps/sbase_test.c
