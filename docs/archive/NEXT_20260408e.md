## Starting from v0.4.69

## Completed in v0.4.69

* **ELF buffer increased to 8MB.** The static elf_buf in aios_root.c
  was 1MB, limiting the maximum executable size. Increased to 8MB to
  support larger binaries like zsh (~6MB). The buffer is in root task
  BSS (not from the VKA pool). RAM cost: ~7MB additional (478MB
  remaining from 2GB).
  Files: src/aios_root.c, include/aios/root_shared.h

* **Morecore pool increased to 4MB.** The musl libc morecore area
  (used for brk/sbrk and mmap MAP_ANONYMOUS) was 1MB per child process.
  Increased to 4MB via CONFIG_LIB_SEL4_MUSLC_SYS_MORECORE_BYTES.
  This allows programs like tcc and zsh to malloc larger internal
  data structures without running out of heap. Required full rebuild.
  Files: deps/seL4_libs/libsel4muslcsys/CMakeLists.txt

* **termios support (TCGETS/TCSETS).** Programs can now call
  tcgetattr() and tcsetattr() on terminal file descriptors. The POSIX
  shim maintains a local struct termios and syncs ICANON and ECHO
  flags to the tty_server via existing TTY_IOCTL ops. TCGETS (0x5401)
  returns current attributes. TCSETS/TCSETSW/TCSETSF (0x5402-0x5404)
  set attributes and apply RAW/COOKED + ECHO ON/OFF to the tty_server.
  This is the foundation for zsh ZLE (line editor) and any
  terminal-aware program.
  Files: src/lib/posix_misc.c, include/aios/tty.h

* **bump-patch.sh and bump-minor.sh fixed.** Use sed -i.bak + rm
  instead of sed -i '' which failed under zsh invocation.
  Files: scripts/bump-patch.sh, scripts/bump-minor.sh

* **DESIGN_TCC.md written.** Design document for porting tcc (Tiny C
  Compiler) to AIOS. ONE_SOURCE cross-compile, AArch64 backend,
  ~250KB binary. Key prerequisite: headers + libraries on disk.
  Milestone: compile and execute C programs inside AIOS.

* **DESIGN_ZSH.md written.** Design document for porting zsh as
  alternative login shell. Three phases: script mode (no AIOS changes),
  interactive (needs termios -- now done), job control (future).
  Per-user shell selection via /etc/passwd pw_shell.

## Lessons Learned in v0.4.69

* **mmap MAP_ANONYMOUS already worked.** The seL4 muslcsys library
  provides mmap via a static morecore pool. The pool was simply too
  small (1MB). No code changes needed -- just increase the cmake
  config value. Full rebuild required since it is a library config.

* **ELF buffer and morecore are separate pools.** elf_buf is in root
  task BSS (used by exec_server to load ELFs). Morecore is per-child
  process (used by musl malloc/mmap). Increasing one does not affect
  the other.

* **termios can be client-side.** The termios struct lives in the POSIX
  shim, not the tty_server. TCGETS returns the local struct. TCSETS
  updates it and sends existing TTY_IOCTL ops to sync the server.
  This avoids packing 60 bytes into message registers and reuses
  the existing RAW/COOKED/ECHO IPC.

* **Full rebuild wipes sbase + dash.** rm -rf build-04 deletes
  everything including cross-compiled sbase tools and dash. Must
  run build_sbase.py + aios-cc dash rebuild after any full rebuild.
