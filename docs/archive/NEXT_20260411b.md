### UART IRQ freeze fix

- QEMU PL011 ICR write clears RX interrupt flag even when data
  remains in the FIFO. QEMU only reasserts the flag on NEW data
  via pl011_put_fifo(), not when data is already present.
- When >64 chars pasted, the drain loop exits at 64, ICR clears
  the flag, seL4_Wait blocks forever, terminal freezes.
- Fix: removed ICR write (unnecessary for level-triggered RX),
  check RXFE (FIFO empty) before seL4_Wait. If FIFO has data,
  yield instead of wait to drain on the next iteration.
- File: src/aios_root.c

### Large file open fix

- fetch_file() returns at most ~900 bytes via IPC message registers.
  aios_sys_open set f->size to the buffered amount (~900), not the
  actual file size. The FS_PREAD on-demand read path only activates
  when f->size > sizeof(f->data), so large files (libc.a at 3.9MB)
  were truncated to ~900 bytes.
- Fix: use fetch_stat() after open to get real file size. The
  FS_PREAD path now activates correctly for large files.
- File: src/lib/posix_file.c

## Bugs Found and Fixed

- **UART IRQ freeze on paste**: QEMU PL011 ICR clears RX flag even
  with data in FIFO. Fix: skip ICR, check RXFE before seL4_Wait.

- **Large file truncation**: f->size set to buffered bytes (~900)
  instead of actual file size. TCC got truncated libc.a. Fix: use
  fetch_stat for real size.

- **TCC CONFIG_TCC_PREDEFS=0**: tcc searched for tccdefs.h at
  runtime (not on disk). Fix: set to 1 for compiled-in definitions.

- **fs_ls_total race**: static variable corrupted by interleaved
  multi-round LS from concurrent clients. Fix: make local.

- **Unchecked seL4 capability ops**: Mint/Copy/ReadRegisters/
  WriteRegisters could fail silently. Fix: check all returns.

## Lessons Learned

- **QEMU PL011 is not real PL011**: Real PL011 RX interrupt is
  level-triggered (set while FIFO non-empty). QEMU only sets the
  flag in pl011_put_fifo() (edge-like). Writing ICR clears it
  regardless of FIFO state. Always check RXFE before sleeping.

- **IPC MR limit truncates large files**: seL4 message registers
  hold ~900 bytes. Any file opened via FS_CAT is limited to this.
  Large files need FS_PREAD (positioned read) for on-demand access.
  The open path must use stat to get the true file size.

- **Static variables in server threads are global state**: Even in
  a single-threaded server, static locals persist across clients.
  If a protocol requires multi-round state (like paginated LS),
  either make it stateless (regenerate each call) or use per-client
  state tracking.
