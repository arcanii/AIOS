## Completed in v0.4.56

* ext2 write-through block cache: 8-entry round-robin in ext2.c.
  read_block() checks cache before disk; write_block() populates
  cache after disk write. Fixes write-then-read bug (echo > file
  then cat file now works). ~8KB static memory, zero API changes.
* ext2 overwrite semantics: ext2_vfs_create() now unlinks any
  existing entry before creating. Prevents duplicate dir entries
  on repeated writes to same file. POSIX-correct behavior.

  stdout/stderr only). Uses same PIPE_WRITE IPC chunking protocol.

  Added pipe_read check before falling through to data buffer path.

  scatter/gather I/O, file descriptors (dup/dup3/fcntl), access

  /chdir/rmdir), signals (sigaction/sigprocmask/sigpending), fork/wait,

  Result: 74 PASS, 0 FAIL, 5 SKIP (pthreads -- thread_ep not in

## Priority 1: Thread Capability Delegation

posix_verify skips 5 pthread tests because exec_thread does not
propagate thread_ep into child CSpace. Fix: add thread_ep to the
cap array passed during ELF loading, update __wrap_main to parse it.
Would bring posix_verify to 79/79.
Files: src/servers/exec_server.c, src/lib/aios_posix.c

## Priority 2: ext2 Directory Cache (unlink verification)

ext2_unlink writes the updated directory block, but ext2_lookup
may still find the entry if the unlinked block is not in the
same cache slot. Current cache fixes CREATE path; verify UNLINK
path also works via cache. Add test to posix_verify.
Files: src/ext2.c
