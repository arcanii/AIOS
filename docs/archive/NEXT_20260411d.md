# AIOS Next Steps -- 2026-04-11d

Starting from **v0.4.77**, building to **v0.4.78**.

## Completed in v0.4.78

### ext2 block/inode freeing on unlink

- Implemented ext2_free_block() -- clears bit in block bitmap, cache invalidate
- Implemented ext2_free_inode() -- clears bit in inode bitmap, guards reserved inodes 1-10
- ext2_unlink() now frees all data blocks: direct (0-11), single-indirect, double-indirect
- Zeroes victim inode on disk before freeing inode bitmap slot
- Previously: unlink removed directory entry but leaked blocks and inodes permanently
- BGDT free count updates on both alloc and free (bg_free_blocks_count, bg_free_inodes_count)
- Both ext2_alloc_block/inode and ext2_free_block/inode now maintain consistent group descriptor counts

### Linux compatibility layer (posix_compat.c)

- New module: src/lib/posix_compat.c with 10 Linux-specific syscalls
- ppoll (NR 73): returns 0 (timeout stub, safe for poll-based programs)
- pselect6 (NR 72): returns 0 (timeout stub)
- getrandom (NR 278): splitmix64 PRNG seeded from AArch64 CNTPCT counter
- prlimit64 (NR 261): reports NOFILE=32, 8MB soft limit for others
- prctl (NR 167): accepts silently (PR_SET_NAME etc.)
- getrlimit (NR 163): same defaults as prlimit64
- setrlimit (NR 164): accepts silently
- sysinfo (NR 179): real uptime from ARM counter, 2GB total RAM
- getrusage (NR 165): zeroed struct stub
- membarrier (NR 283): returns 0 (success)

### Virtual device files

- /dev/urandom: read returns splitmix64 PRNG bytes (up to 4096 per read)
- /dev/random: same as /dev/urandom (no entropy pool distinction)
- /dev/zero: read returns zero bytes (up to 4096 per read)
- All three use is_devnull=1 with path stored for device type dispatch
- Write to any device returns count (sink, same as /dev/null)
- Same open pattern in both sys_open and sys_openat

### readlinkat /proc/self/exe

- __wrap_main now stores program path in aios_progpath[128] global
- readlinkat checks for "/proc/self/exe" path match
- Returns stored program path (e.g. "/bin/dash", "/bin/echo")
- Programs that discover their own path via /proc/self/exe now work

### Expanded procfs

- /proc/cpuinfo: per-core processor index + model name from DTB hw_info
- /proc/stat: aggregate cpu line + per-core cpu0..cpuN lines (zero counters)
- /proc/loadavg: static 0.00 0.00 0.00 load average stub
- Disambiguated existing handlers: log (path[2]==g) vs loadavg (path[2]==a),
  status (path[4]==u) vs stat (path[4]==0)
- Directory listing updated: 11 entries (was 8)

### Process table limit increases

- MAX_ZOMBIES: 8 -> 16 (prevents silent exit status loss)
- MAX_WAIT_PENDING: 4 -> 8 (supports more concurrent waitpid calls)
- MAX_PIPES: 8 -> 16 (supports heavier pipeline workloads)
