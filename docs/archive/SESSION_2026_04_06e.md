## Completed

### 1. &&/|| Chains with Pipes (Priority 1)

* run_pipeline() now returns last child exit status (POSIX: pipeline
  status = exit status of final command)
* Chain handler detects | in left side, calls run_pipeline() for piped
  left-side commands instead of do_exec()
* Removed dead lcmd[512] CWD construction from chain handler
* Verified: echo hello | cat && echo world | cat
* Verified: echo hello | grep xxx || echo fallback | cat
* Verified: true && echo yes | cat
* Verified: false || echo recovered | cat

### 2. Remove exec_ep (Priority 2)

* Removed exec_ep variable, initialization, argv[2] parse
* Removed dead cmd[512] CWD construction (was built but never consumed)
* Updated header comment
* mini_shell no longer references exec_ep -- pure fork+exec

### 3. Getty/Shell Separation (Priority 3)

* Created src/apps/getty.c: banner + login loop + MOTD + fork+exec
  mini_shell + waitpid loop
* Converted mini_shell.c from AiosChildApp to AiosPosixApp
  (__wrap_main handles cap extraction + uid/gid)
* Moved do_login(), read_password(), banner, MOTD display to getty
* exit/logout at su_depth 0 calls _exit(0), returning to getty waitpid
* Getty re-loops to login prompt on shell exit
* New PIPE_SET_IDENTITY IPC (label 74): getty updates active_proc
  uid/gid after login so fork+exec propagates identity correctly
* Session identity in mini_shell derived from getuid()/getpwuid()
  (no more manual argv parsing of session credentials)

## Key Bug Found and Fixed

### run_pipeline return value corrupted env_get()
Script A used text substitution to replace "return 0;" with
"return last_status;" but the pattern matched inside env_get()
(which also returns 0). env_get() returned an undeclared identifier,
causing a compile error. Fixed with line-number-specific patching.
