## Key Bugs Found and Fixed

### 1. Pipe double-create (root cause of slot exhaustion)
Old code created server-side pipes via raw IPC, then run_pipeline created
its own via POSIX pipe(). First set leaked. Fix: removed old loop.

### 2. cd falling through to exec
cd builtin set CWD but had no continue, so shell tried to exec "cd".
Fix: added continue.

### 3. pwd showing "/" after cd
PIPE_EXEC hardcoded CWD to "/". Child never inherited parent CWD.
Fix: pack CWD in PIPE_EXEC buffer, extract server-side.

### 4. Redirect freeze on exec failure
do_exec_piped returned -1 but shell still called PIPE_READ, blocking
forever. Fix: check return, PIPE_CLOSE on failure.

### 5. Redirect freeze on child exit (exec_server path)
Children spawned via exec_server fault on exec_server EP, not pipe_ep.
handle_child_fault never runs. Fix: rewrote redirects to fork+dup2+exec.

### 6. Input redirect missing EOF
Parent wrote data and closed local fd, but server-side pipe kept
write_closed=0. Fix: explicit PIPE_CLOSE_WRITE IPC after writing.
