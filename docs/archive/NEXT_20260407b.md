# AIOS Next Steps -- 2026-04-07b

## Starting from v0.4.52

## Priority 1: Pipeline Error Recovery

run_pipeline has no cleanup on mid-pipeline fork failure. If fork fails
for segment N of M, earlier children block forever writing to a pipe
whose reader was never spawned.

Fix:
* On fork failure: send SIGKILL equivalent (PIPE_KILL) to all already-
  forked children in the pipeline
* Close all pipe fds (both read and write ends)
* Destroy server-side pipe buffers via PIPE_CLOSE
* waitpid for killed children to prevent zombies
* Return -1 to indicate pipeline failure

## Priority 2: Multi-Chain Support (&&/|| recursion)

Current chain handler only splits at the first && or ||.
"true && echo a && echo b" drops the second chain.

Options:
* Iterative: scan right-to-left for last && or ||, split there,
  recurse on left side. Gives correct left-to-right evaluation
  with short-circuit semantics.
* Or: preprocess line into chain_segment[] array with operators,
  evaluate left-to-right with early exit.

Both approaches must handle mixed operators:
"cmd1 && cmd2 || cmd3 && cmd4"

## Priority 3: Getty Robustness

* Fork failure: retry 3x with 500ms sleep, then fall back to running
  mini_shell directly in getty process (degraded mode)
* Log WIFSIGNALED when child shell dies from fault
* Ctrl-C handling in read_password and login input

## Priority 4: PIPE_EXEC_WAIT Deadlock Recovery

If child crashes between PIPE_FORK and PIPE_EXEC (during ELF load),
exec_done[ci] never set, parent blocks in PIPE_EXEC_WAIT forever.

Fix: handle_child_fault should call check_exec_wait(child_idx) to
unblock any parent waiting on the crashed child. Already have the
child_idx from the fault handler -- just need the call.

## Priority 5: Static Buffer Cleanup

do_exec() and do_exec_bg() use static buffers for argv construction.
Move to stack-local to prevent corruption under future concurrency.

## Priority 6: Signal Infrastructure (v0.5.x prep)

Required for proper job control:
* SIGCHLD delivery to parent on child exit
* SIGINT (Ctrl-C) delivery to foreground process group
* SIGTERM for kill(pid, SIGTERM)
* signal()/sigaction() in aios_posix
* Process groups (setpgid/getpgid)

## Priority 7: Modularize aios_root.c

aios_root.c is 674 lines. Extract:
* boot_init.c: allocator, SMP, driver init
* boot_spawn.c: CPIO load, service spawn
* Keep aios_root.c as orchestrator (<200 lines)
