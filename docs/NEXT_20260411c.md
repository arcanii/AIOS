## Completed in v0.4.77

### DTB-based hardware discovery

- Parse device tree blob from seL4 bootinfo extras via libfdt
- Discovers UART address + IRQ, virtio MMIO base, fw_cfg, CPU
  count + compatible string, memory base + size
- Falls back to QEMU virt defaults if DTB absent or malformed
- Boot prints `[hw]` inventory lines with discovered hardware
- Replaces hardcoded UART0_PADDR, UART0_IRQ in aios_root.c
- Replaces hardcoded VIRTIO_BASE_ADDR in boot_fs_init.c
- Replaces hardcoded fw_cfg address in boot_display_init.c
- New files: include/aios/hw_info.h, src/boot/boot_dtb.c

### Dynamic /proc/version from DTB

- /proc/version now reports CPU compatible string and core count
  discovered from the device tree instead of hardcoded values
- Example: "AIOS 0.4.x (seL4 15.0.0, arm,cortex-a53, 4-core SMP)"

### Procfs zero-size file open fix

- procfs_stat returns size=0 for dynamic files (content generated
  on read). The openat handler used fetch_stat to get file size,
  then skipped reading when size=0.
- Fix: when stat returns size=0, fall back to fetch_file (FS_CAT)
  which reads the actual content from procfs.
- Affected: cat /proc/version, cat /proc/meminfo, etc.
- Root cause: v0.4.72 openat rewrite used fetch_stat + fetch_pread
  instead of fetch_file, breaking procfs reads. Unnoticed because
  posix_verify uses uname IPC (not procfs file reads).
- IMPORTANT: sbase/dash must be rebuilt after posix_file.c changes
  (ninja only rebuilds libaios_posix.a, not user binaries).

### Procfs directory listing fix

- procfs_stat returned regular-file mode (0100444) for all paths
  including /proc itself. ls /proc showed it as a file, not dir.
- Fix: return directory mode (040555) when path is root "/".
- ls /proc now lists: PIDs, version, meminfo, mounts, uptime, etc.

## Bugs Found and Fixed

- **Procfs reads broken since v0.4.72**: openat used fetch_stat
  which returns 0 for procfs. Read loop skipped. Fix: fetch_file
  fallback for zero-size stat results.

- **ls /proc showed as file**: procfs_stat always returned regular
  file mode. Fix: directory mode for root path.

## Lessons Learned

- **sbase rebuild required for POSIX shim changes**: libaios_posix.a
  is statically linked into every user binary. ninja recompiles the
  library but does NOT rebuild sbase/dash. Always run build_apps.py
  (not just ninja) when posix_*.c files change.

- **procfs stat cannot report size**: procfs content is generated
  dynamically. stat must return 0 for size. The open path must
  handle this by reading content directly via FS_CAT instead of
  relying on stat size for demand-paging.

- **DTB is in bootinfo extras**: seL4 copies the DTB into the
  rootserver bootinfo extras buffer with header type
  SEL4_BOOTINFO_HEADER_FDT. Access via simple_get_extended_bootinfo.
  Must skip the 16-byte seL4 header before passing to fdt_check_header.

- **GIC interrupt format in DTB**: ARM GIC interrupts have 3 cells:
  <type irq_num flags>. Type 0 = SPI, actual IRQ = irq_num + 32.
