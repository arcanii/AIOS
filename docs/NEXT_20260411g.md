## Completed in v0.4.81

### TCP stream socket read/write

- `posix_file.c`: `read()` on socket fds routes via `NET_RECVFROM` IPC to net_server
- `posix_file.c`: `write()` on socket fds routes via `NET_SENDTO` IPC to net_server
- Reuses existing IPC labels -- no new protocol needed (TCP SENDTO ignores dst_ip/port for ESTAB sockets)
- Data packed into MR registers (up to 900 bytes per call), chunked loop for larger writes

### TCP circular receive buffer

- `net_server.c`: `SOCK_RX_BUF_SZ` increased from 2KB to 4KB (power-of-2 for bitmask)
- New fields: `rx_head`, `rx_tail` (ring buffer positions), `rx_eof` (FIN received flag)
- TCP RECVFROM handler: three paths -- data available (ring drain), EOF (return 0), empty (SaveCaller block)
- UDP path unchanged (single datagram with `rxlen`)
- FIN handling: `rx_eof=1` set before state change, reader drains buffer then gets EOF

### Virtio-net dedicated IRQ handler

- `boot_net_init.c`: binds virtio-net IRQ (48 + slot) to `net_drv_ntfn_cap` via `seL4_IRQHandler_SetNotification`
- `net_driver.c`: ACKs both virtio MMIO ISR and seL4 IRQ handler after processing packets
- `aios_root.c`: removed virtio-net ISR polling from UART event loop (was piggybacking on key press)
- **Root cause fixed**: networking previously only worked when a key was pressed because the UART IRQ handler was the only thing polling the virtio ISR register

### Event-driven net_server

- `net_server.c`: `seL4_NBRecv` (busy-poll) replaced with `seL4_Recv` (blocking)
- Bound notification (`net_srv_ntfn_cap`) wakes server for packets; IPC endpoint wakes for socket API
- Removed `seL4_Yield` and `poll_count` busy-poll artifacts
- **Root cause fixed**: busy-polling with NBRecv+Yield starved QEMU SLIRP event loop, preventing packet delivery to virtio-net

### Socket syscall stubs

- `posix_net.c`: `getsockname()`, `getpeername()`, `getsockopt()` stubs

## Architecture After v0.4.81

### Net Server Event Model

```
net_driver (interrupt-driven):
  seL4_Wait(drv_ntfn)          <-- virtio-net IRQ wakes this
  drain virtio used ring -> rx_ring
  seL4_Signal(net_srv_ntfn)    <-- wake server
  ACK virtio ISR + seL4 IRQ

net_server (event-driven):
  drain rx_ring -> net_handle_packet
  seL4_Recv(ep)                <-- blocks until notification OR IPC
    notification (packets) -> loop back to drain rx_ring
    IPC (socket API) -> process + reply
```

### Socket fd Data Path

```
User program: read(sockfd) / write(sockfd)
  -> posix_file.c: is_socket check
  -> seL4_Call(net_ep, NET_RECVFROM/NET_SENDTO)
  -> net_server: TCP circular buffer read / TCP segment send
  -> seL4_Reply with data / ACK
```
