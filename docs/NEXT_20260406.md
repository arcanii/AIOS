# AIOS Next Steps — 2026-04-06

## Starting from v0.4.50

## 1. Convert Pipe Operator to fork+pipe2+dup2+exec

### Current State
ls | grep bin uses legacy do_exec_piped() which sends CWD-encoded pipe
IDs via EXEC_RUN IPC to exec_thread. This requires exec_ep.

### Target
Replace with pure POSIX: pipe2() -> fork() -> dup2() -> execv()
Multi-pipe: chain pipe2 calls for cmd1 | cmd2 | cmd3.
Wait for last child in pipeline.

### Changes Needed
- New do_exec_piped_fork() using pipe2+fork+dup2+exec
- Remove old do_exec_piped() and do_exec_piped_ipc()

## 2. Getty/Shell Separation

### Architecture
root spawns getty (CPIO, 5 caps: ser, fs, exec, auth, pipe)
  getty: banner + login loop
  getty: EXEC_RUN mini_shell
  getty: blocks until shell exits
  getty: loops back to login

mini_shell: pure command interpreter (no login code)
  exit/logout: return 0 -> getty resumes

### Changes
- Create src/apps/getty.c (ChildApp)
- Strip login/banner/MOTD from mini_shell
- Root spawns getty instead of mini_shell

## 3. Remove exec_ep from mini_shell

After pipe conversion, mini_shell has zero EXEC_RUN calls.
Convert to PosixApp (--wrap=main). Any shell works identically.

## 4. Signal Infrastructure (Future)

- SIGCHLD, SIGINT, SIGTERM
- signal()/sigaction() in aios_posix
