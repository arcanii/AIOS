## Features Implemented

### 1. Linux Build Environment

Full AIOS build working on Ubuntu 25.10 (x86_64, GCC 15.2, Python 3.13).

**Key setup requirements discovered:**
- Python venv needed for seL4 build deps (pyfdt, pyyaml, lxml, ply, pyelftools, libarchive-c)
- `g++-aarch64-linux-gnu` required (not just gcc)
- `libxml2-utils` required for xmllint
- `python3.13-venv` package needed before venv creation
- cmake-tool symlink must use `../../deps/` (two levels up from tools/seL4/)
- build_number.h must be created manually on first build

**Scripts created:**
- `scripts/setup-linux.py` — automated full environment setup
- `scripts/build_dash.py` — cross-compile dash with correct flags
- `scripts/gen-builtins-def.py` — preprocess builtins.def.in (JOBS=0, SMALL=1)
- `scripts/qemu-boot.py` — QEMU launcher with --net/--display options

### 2. Dash Build on Linux

**Problems solved:**
- `builtins.def` missing: generated from `builtins.def.in` via Python preprocessor
- `token.h` missing: generated by `sh mktokens` (must run first)
- `killpg` conflict: musl declares it, dash system.h provides fallback -- add `HAVE_KILLPG 1`
- `wait3` invisible: `sys/wait.h` include guarded by `#ifdef BSD` in jobs.c -- add `BSD 1` to config.h
- `wait3` function itself: replaced with `waitpid(-1, status, flags)` in jobs.c

## Key Learnings

### Linux vs macOS Build Differences

- sbase path: `~/Source/github/sbase` not `~/Desktop/github_repos/sbase`
- dash path: `~/Source/github/dash` not `~/Desktop/github_repos/dash`
- build_sbase.py needs `--sbase` flag on Linux
- No Homebrew; use apt packages directly
- python3-venv is a separate package on Ubuntu

### seL4 Python Dependencies (complete list)

pyfdt, pyyaml, jinja2, jsonschema, lxml, ply, pyelftools, libarchive-c

These must be in the PATH when running cmake and ninja. Use the .venv.

### Dash Build Dependencies (complete list for Linux)

Generated files (in order): token.h, syntax.c/h, nodes.c/h, init.c,
signames.c, builtins.def, builtins.c/h, config.h
