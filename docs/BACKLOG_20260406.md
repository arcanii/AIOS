# AIOS Backlog -- 2026-04-06

## Critical

### 1. Getty fork failure = dead system
getty.c: if fork() fails, it prints error and loops to login. Repeated
failure (proc table full) means user logs in, sees error, loops forever.
Fix: retry with backoff (3 attempts, 500ms sleep). On persistent failure,
fall back to running mini_shell directly in getty process (degraded single-
process mode rather than dead system).
Files: src/apps/getty.c

## High

### 2. run_pipeline: no cleanup on mid-pipeline fork failure
If fork fails for segment N of M, earlier children are already running
and writing to pipes. Code sets child_pids[si] = -1 and continues, but
earlier children may block forever writing to a pipe whose reader was
never spawned.
Fix: on any fork failure, kill+reap already-forked children, close all
pipe fds, destroy server-side pipe buffers. Return -1 immediately.
Files: src/apps/mini_shell.c (run_pipeline)

### 3. PIPE_EXEC_WAIT: no timeout / deadlock recovery
If child crashes between PIPE_FORK and PIPE_EXEC (during ELF load),
exec_done[ci] is never set, check_exec_wait is never called, and the
parent shell blocks forever in PIPE_EXEC_WAIT.
Fix: handle_child_fault should call check_exec_wait for the faulted
child index (unblocking any waiting parent). Alternatively, add a
timeout mechanism via seL4 notification + timer.
Files: src/servers/pipe_server.c

## Medium

### 4. Static buffers in do_exec / do_exec_bg
do_exec() uses static char argv0[64] and argbuf[512]. do_exec_bg()
uses static char bg_argv0[64] and bg_argbuf[512]. These would corrupt
under concurrent or reentrant calling. Not a bug today (single-threaded
shell), but will bite when signals or job control are added.
Fix: move all static buffers to stack-local.
Files: src/apps/mini_shell.c

### 5. Getty: no fault/signal logging for child shell
waitpid() returns but we only check WIFEXITED. If shell dies from a
fault (WIFSIGNALED), getty silently loops back to login with no
diagnostic. Helpful for debugging.
Fix: check WIFSIGNALED(status) and log signal number before re-looping.
Files: src/apps/getty.c

## Low

### 7. read_password: no Ctrl-C handling
In getty login input and read_password(), Ctrl-C is silently ignored.
User cannot abort a login attempt; must type 3 bad passwords.
Fix: detect Ctrl-C (0x03), return failure immediately from do_login
attempt, loop back to fresh login prompt.
Files: src/apps/getty.c

## Future

### 8. Shared-memory pipes
Current IPC pipes: pipe_server round-trip per chunk (~900 bytes max).
Shared-memory: vka_alloc_frame mapped into both writer and reader
VSpaces. Ring buffer with atomic head/tail. Zero IPC for data transfer.
pipe_server only handles lifecycle (create/destroy).

### 10. Dynamic ELF buffer
elf_buf is static 1MB. Large programs (gcc, etc.) will not load.
Fix: vka_alloc_frame for variable-size ELF loading.
