## Priority 1: ext2 Directory Cache (write-then-read)

After ext2_create_file() writes a new directory entry and file data,
ext2_lookup() cannot find the new entry in the same session. Root cause:
ext2_lookup reads the directory block from disk, but ext2_create_file
may have written the entry to a block that the read path re-fetches
from stale data.

Fix options:
  a) Verify ext2_write_block flushes synchronously to virtio-blk.
  b) Add a small directory-block write-through cache.
  c) Re-read parent directory inode after create.

Impact: unlocks posix_verify 42/42, cp create-then-verify, tee+cat.
Files: src/ext2.c

## Priority 2: Getty Robustness

* Fork failure retry with backoff (3 attempts, 500ms sleep)
* On persistent failure, fall back to mini_shell in-process (degraded mode)
* WIFSIGNALED logging on child crash
* Ctrl-C (0x03) handling in login input and read_password()
Files: src/apps/getty.c

## Priority 3: Signal Phase 3 -- Cross-Process Delivery

Current aios_sig_check() only runs on kill/tgkill return paths.
For cross-process signal delivery (parent sends SIGUSR1 to child),
the child needs to check sig_pending on other syscall return paths
(read, write, nanosleep). Add check points to blocking syscalls.
Files: src/lib/aios_posix.c

## Priority 4: Pipeline Cleanup on Mid-Fork Failure

If fork fails for segment N of M, earlier children are already running.
Fix: on any fork failure, SIGKILL+reap already-forked children, close
all pipe fds, destroy server-side pipe buffers. Return -1 immediately.
Files: src/apps/mini_shell.c (run_pipeline)

## Priority 5: Static Buffer Cleanup

do_exec() and do_exec_bg() use static buffers for argv0 and argbuf.
Move to stack-local to prevent corruption under future reentrancy.
Files: src/apps/mini_shell.c

## Priority 6: getconf (last sbase tool)

getconf.c requires a generated getconf.h with system configuration
values. Write scripts/gen_getconf.py to produce a minimal header with
AIOS-appropriate values (PAGE_SIZE, NGROUPS_MAX, etc.).
Would bring sbase count to 99/99.

## Priority 7. Shared-memory pipes
Current IPC pipes: pipe_server round-trip per chunk (~900 bytes max). Shared-memory: vka_alloc_frame mapped into both writer and reader VSpaces. Ring buffer with atomic head/tail. Zero IPC for data transfer. pipe_server only handles lifecycle (create/destroy).

## Priority 8. Dynamic ELF buffer
elf_buf is static 1MB. Large programs (gcc, etc.) will not load. Fix: vka_alloc_frame for variable-size ELF loading.
