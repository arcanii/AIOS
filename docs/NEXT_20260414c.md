## Features Implemented

### 1. HDMI Display from Root Task (display_vc.c rewrite)

**Problem:** The diagnostic stub initialized a 1024x768 framebuffer via
VideoCore mailbox and stored its address in fb_info at phys 0x3A000000.
The root task could not map this because seL4 would zero the data on retype.

**Solution:** Extended the DTS reserved-memory from 0x3B400000 to 0x3A000000
(overlay-rpi4.dts + overlay-rpi4-4gb.dts). This creates device untypeds
(NOT zeroed on retype) for the range, preserving the diagnostic stub data.

display_vc.c rewritten with two-phase init:
- Phase A: Map fb_info at 0x3A000000, read FB address, map framebuffer pages
- Phase B: Fallback to full VC mailbox protocol (existing code)

Phase A works: fb_info read successfully, 768 pages mapped, fb_console active.

### 2. SD Card Driver (blk_emmc.c -- Phase 1)

BCM2711 EMMC2 SDHCI driver with PIO-mode single-block reads.

**Init sequence:** Controller reset, set 400kHz clock, CMD0 (idle),
CMD8 (SDv2 detect), ACMD41 loop (card ready), CMD2 (CID), CMD3 (RCA),
CMD7 (select), set 25MHz clock, ACMD6 (4-bit bus).

**Sector read:** CMD17 with PIO -- 128 x 32-bit reads from SDHCI buffer port.

**MBR parsing:** Reads sector 0, finds type 0x83 (Linux/ext2) partition,
stores start LBA as offset. plat_blk_read() adds partition offset.

**Result:** ext2 filesystem mounted from SD card partition 2. All 130+
programs (sbase, dash, AIOS apps) loaded and executed.

### 3. DTB Parser Fix (boot_dtb.c)

**Problem:** fdt_read_reg64() always read 64 bits, but RPi4 soc nodes use
#address-cells=1 (32-bit). UART address read as 0x7e20100000000200 (garbage).

**Fix:** Added fdt_read_reg_node() that checks parent #address-cells via
fdt_parent_offset() + fdt_address_cells(). Reads 32 or 64 bits accordingly.

**VC-to-ARM translation:** RPi4 DTB has VC bus addresses (0x7Exxxxxx).
ARM physical addresses are at +0x80000000 (0xFExxxxxx). Translation added
in fdt_read_reg_node() for addresses in range 0x7C000000-0x7FFFFFFF.

**parse_memory fix:** Handles variable #address-cells/#size-cells, sums
all memory regions. RPi4 4GB now correctly reports 3936 MB.

### 4. Mini UART Serial Input (aios_root.c)

**Problem:** Root task mapped PL011 (0xFE201000) for UART I/O. But without
overlays/disable-bt.dtbo on the SD card, GPIO 14/15 are ALT5 = mini UART,
NOT PL011. The seL4 kernel also uses mini UART (bcm2835-aux-uart) for
debug output.

**Root cause chain:**
1. mksdcard.py does not include overlays/ directory on FAT32 partition
2. Without disable-bt.dtbo, firmware keeps GPIO 14/15 as mini UART
3. seL4 DTS has serial1 = mini UART, kernel uses it for putchar
4. PL011 at 0xFE201000 is connected to Bluetooth module, not GPIO header
5. Root task mapping PL011 reads/writes to wrong hardware

**Fix:** Root task now maps mini UART (AUX block at 0xFE215000) on RPi4.
Uses MU_IO (offset 0x40) and MU_LSR (offset 0x54) for data/status.
Kernel marks this device frame userAvailable=true.

Polling mode (no IRQ for mini UART in Phase 1).

### 5. Elfloader Cleanup

Removed ~30 diag_val/diag_bar debug calls from elfloader common.c and
sys_boot.c. Re-added #define printf(...) ((void)0) for BCM2711 (mini UART
in elfloader hangs CPU bus -- elfloader uses serial1 = mini UART).

## Key Learnings

### RPi4 UART Architecture (without disable-bt overlay)

- GPIO 14/15 = ALT5 = mini UART (NOT PL011)
- seL4 kernel uses serial1 = mini UART via bcm2835-aux-uart driver
- PL011 (0xFE201000) is connected to Bluetooth module, not GPIO header
- Mini UART AUX block at 0xFE215000, registers at offset 0x40
- Mini UART register layout differs from PL011 (MU_IO, MU_LSR vs DR, FR)

### seL4 Device Untypeds and Memory Types

- reserved-memory with no-map removes region from RAM untypeds
- Gaps between ndks_boot.reserved[] entries become device untypeds
- Device untypeds are NOT zeroed on retype (seL4 boot.c:696 isDeviceCap check)
- This allows preserving data written by pre-seL4 code (diagnostic stub)

### BCM2711 Address Translation

- DTB has VC bus addresses (0x7Exxxxxx) for soc peripherals
- ARM physical = VC bus + 0x80000000 (0xFExxxxxx)
- overlay-rpi4-address-mapping.dts only translates a few devices (has TODO)
- DTB parser applies translation for addresses in 0x7C-0x7F range

### SD Card Partition Layout

- Partition 1: FAT32 (boot firmware, kernel8.img, config.txt)
- Partition 2: ext2 (AIOS system disk, created by mkdisk.py)
- blk_emmc.c parses MBR, finds type 0x83, adds LBA offset to all reads
