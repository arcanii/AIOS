### SSH server Phase 2: key exchange

- `ssh_crypto.c`: mbedTLS wrappers for CTR-DRBG, ECDSA P-256, x25519 ECDH
- ECDSA P-256 host key generated at boot (in-memory, regenerated each restart)
- x25519 ephemeral key exchange (curve25519-sha256, RFC 8731)
- Exchange hash H computed per RFC 4253 section 8 (SHA-256 of V_C, V_S, I_C, I_S, K_S, Q_C, Q_S, K)
- ECDSA signature: signs SHA-256(H) per OpenSSH convention (not H directly)
- x25519 shared secret exported as RFC 7748 LE bytes (matching OpenSSH sshbuf_put_bignum2_bytes convention)
- Session key derivation: SHA-256(K_mpint || H || letter || session_id) for letters A-F
- Strict KEX: sequence numbers reset after KEXINIT exchange
- SSH_MSG_NEWKEYS sent and received -- key exchange fully verified with OpenSSH 10.3 (Homebrew/OpenSSL 3.6.1)

## Build Commands

### Build libmbedcrypto.a (one-time, ~2s)

```
python3 /tmp/script_e2_build_mbedcrypto.py
```

### Build sshd

```
cd ~/Desktop/github_repos/AIOS
GCC_INC=$(aarch64-linux-gnu-gcc -print-file-name=include)
./scripts/aios-cc \
    src/ssh/sshd_main.c src/ssh/ssh_transport.c \
    src/ssh/ssh_kex.c src/ssh/ssh_crypto.c \
    -I src/ssh -I ~/Desktop/github_repos/mbedtls/include \
    -isystem "$GCC_INC" -DMBEDTLS_ALLOW_PRIVATE_ACCESS \
    build-04/libmbedcrypto.a \
    -o build-04/sbase/sshd
```

### QEMU with networking

```
qemu-system-aarch64 \
    -machine virt,virtualization=on \
    -cpu cortex-a53 -smp 4 -m 2G \
    -nographic -serial mon:stdio \
    -drive file=disk/disk_ext2.img,format=raw,if=none,id=hd0 \
    -device virtio-blk-device,drive=hd0 \
    -device virtio-net-device,netdev=net0 \
    -netdev user,id=net0,hostfwd=tcp::2222-:2222 \
    -kernel build-04/images/aios_root-image-arm-qemu-arm-virt
```
