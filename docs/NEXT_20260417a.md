## Session Result: ZSH Interactive Mode with ZLE

### Goal
Enable ZSH ZLE (Zsh Line Editor) for interactive cursor-based line editing.
This required significant kernel/POSIX infrastructure work beyond just
recompiling ZSH.

### Infrastructure Changes (shared, benefits all programs)

1. **Server-side termios** (tty_server.c)
   - Full termios struct stored server-side (ts_iflag/oflag/cflag/lflag/cc)
   - ISIG flag controls Ctrl-C signal generation
   - TCGETS/TCSETS transfer full termios via 7 IPC message registers
   - TTY_POLL IPC returns available input byte count
   - Raw-mode TTY_READ reads from key_buf (single-char for ZLE)
   - ICRNL (CR->NL) conversion respects termios flag
   - Legacy SET_RAW/COOKED/ECHO_ON/ECHO_OFF update termios flags

2. **Real ppoll/pselect** (posix_compat.c)
   - ppoll: queries TTY_POLL for stdin + is_tty AIOS fds
   - ppoll: probes pipe_server for pipe fds
   - pselect6: scans fd_set for tty fds (any bit position, not just fd 0)
   - Timeout: sleeps up to 50ms then re-checks
   - Replaced stubs that returned 0 immediately

3. **Duped tty fd support** (posix_misc.c, posix_file.c)
   - ioctl: TCGETS/TCSETS/TIOCGWINSZ work for AIOS fds with is_tty flag
   - read: blocking TTY_READ loop for is_tty fds (Yield until data)
   - write: routes is_tty to aios_stdio_write (serial)
   - writev: same routing for is_tty (musl FILE* uses writev)
   - Fixes zsh SHTTY = dup(0) -> tcgetattr/fdopen path

4. **New syscall stubs** (posix_compat.c)
   - rt_sigsuspend (133): returns -EINTR after Yield
   - setitimer (103): returns success, zeroes old value
   - getitimer (102): returns zeroed timer

5. **VT100 termcap library** (src/lib/termcap.c)
   - Hardcoded VT100/xterm capabilities (no termcap database)
   - 60+ string capabilities (cursor, clearing, attributes, keys)
   - 6 numeric (80x24, 8 colors), 6 boolean (am, xn, bs, mi, ms)
   - tgetent/tgetstr/tgetnum/tgetflag/tgoto/tputs/tparm
   - cur_term/setupterm/set_curterm/del_curterm stubs
   - Cross-compiled to libtermcap.a (9KB, 10 symbols)

### ZSH Build Changes

- ZLE enabled: 15 new source files (Zle/*.c) added to build
- bltinmods.list: zsh/zle module registered (autofeatures + boot)
- config.h: HAVE_TGETENT=1, disabled RLIMIT_RTTIME, SYS_RANDOM_H, GETRANDOM
- Compile flag: -DLINKED_XMOD_zshQszle (gates ZLE autoload in zleentry)
- Generated headers: widgets.list, zle_widget.h from iwidgets.list
- random_real.c added (dependency of random.c)
- rlimits.c and sched.c: correct path is Builtins/, not Modules/
- scripts/build_zsh.py: repeatable 55-file cross-compile + link

### Bug Fixes Found During Integration

| Bug | Root cause | Fix |
| --- | --- | --- |
| "bad tcgets: not a tty" | ioctl only checked fd <= 2, not duped tty fds | is_tty_fd check for AIOS fds |
| ZLE never enters read loop | pselect6 only checked fd 0 bit, not SHTTY fd | Scan full fd_set for is_tty fds |
| ZLE prompt invisible | writev missing is_tty routing | Added is_tty check in writev |
| ZLE read returns EOF | TTY_READ returned 0 for empty key_buf | Blocking loop with Yield |
| LINKED_XMOD_zshQszle missing | zleentry compiled without ZLE dispatch | Added -D flag to build |
| syscall 133 spam | rt_sigsuspend not implemented | Stub returns -EINTR |
