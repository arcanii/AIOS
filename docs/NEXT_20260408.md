# AIOS Next Steps -- 2026-04-08

## Starting from v0.4.64

## Completed in v0.4.64

* Allocator pool increased from 1500 to 4000 pages (6 MB to 16 MB).
  Band-aid for dash fork budget (was 18, need 40+).
  Files: src/aios_root.c

* pip_dest cap leak fixed in exec_server.c. Every process exec
  mints a badged pipe_ep but only te_dest and fs_dest were freed
  on cleanup. Both Ctrl-C and normal-exit paths now free pip_dest.
  Files: src/servers/exec_server.c

* /dev/null support. Both open() and openat() detect "/dev/null"
  path before touching VFS. Write/writev return count (discard).
  Read returns 0 (size=0). is_devnull flag in aios_fd_t struct.
  Files: src/lib/posix_file.c, src/lib/posix_internal.h

* dup2 identity case: dup3 now returns fd when oldfd==newfd
  (POSIX dup2 requirement, used heavily by dash fd management).
  Files: src/lib/posix_misc.c

* ioctl TIOCGWINSZ: returns 80x24 terminal size, making isatty()
  work reliably. Also handles TIOCGPGRP (return aios_pid) and
  TIOCSPGRP (stub). Used by dash for interactive detection.
  Files: src/lib/posix_misc.c

* fcntl F_DUPFD and F_DUPFD_CLOEXEC: dash uses fcntl(fd, F_DUPFD, 10)
  to relocate internal fds above 10. Now allocates new aios fd
  and copies source fd state.
  Files: src/lib/posix_misc.c

* setpgid stub: flat process model, always returns 0. Wired as
  __NR_setpgid (154) in syscall install table.
  Files: src/lib/posix_proc.c, src/lib/posix_internal.h, src/lib/aios_posix.c

* dash shell ported and cross-compiled. 468KB AArch64 ELF,
  statically linked. Binary executes on AIOS (dash -c with no arg
  prints usage). stdout for -c "echo hello" not yet visible --
  likely mini_shell quote parsing strips quotes before passing to exec.
  Files: ~/Desktop/github_repos/dash/src/ (external)
  Build: see AI_BRIEFING.md "Build dash" section

## Dash Port Status

### What works
* Binary loads and executes in AIOS VSpace
* dash -c (no arg) prints "dash: 0: -c requires an argument" on stderr
* All 31 source files compile clean with aios-cc
* Links successfully against aios_posix + musl + seL4 runtime

### What needs debugging
* dash -c "echo hello" produces no visible stdout output
  - Likely cause 1: mini_shell quote handling -- single quotes may be
    stripped or split incorrectly before passing to exec_thread
  - Likely cause 2: dash uses its own output buffering (output.c),
    may not flush on exit
  - Test plan: write script to file, run dash /tmp/test.sh
    to bypass mini_shell quote parsing entirely
