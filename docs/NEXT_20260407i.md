# AIOS Next Steps -- 2026-04-07i

## Starting from v0.4.59

## Completed in v0.4.59

* ext2 cache: added cache_invalidate() utility for defense-in-depth.
  write_block() already calls cache_store(), so unlink path was
  correct; cache_invalidate() available for future fsck/format use.
  Verified: echo > file, cat, unlink, cat returns "No such file".
  Files: src/ext2.c

* Getty robustness: fork retry with 3-attempt progressive backoff
  (500ms, 1s, 1.5s) via ARM generic timer. On persistent failure,
  falls back to execv mini_shell in-process (system never locked out).
  WIFSIGNALED/WTERMSIG logging on child crash. Ctrl-C (0x03) handling
  in both username and password input loops -- breaks current attempt,
  returns to fresh login prompt.
  Files: src/apps/getty.c

* Signal Phase 3 -- cross-process delivery: aios_sig_check() now
  called on entry to read() and write() (the two most common blocking
  syscalls in posix_file.c), and periodically during the nanosleep()
  busy-wait loop (posix_time.c). Signals delivered via pipe_server
  are now dispatched even when the target process is blocked in I/O
  or sleeping. signal_test 20/20.
  Files: src/lib/posix_file.c, src/lib/posix_time.c

* Shell comment support: lines starting with # are silently ignored.
  Files: src/apps/mini_shell.c

* posix_rm argv fix: was checking argc < 3 / argv[2] (pre-wrap_main
  indexing); fixed to argc < 2 / argv[1].
  Files: src/apps/posix_rm.c

* Meaningful pseudo-inodes: fstat/fstatat now return st_dev=1 and
  st_ino from djb2 hash of the path string. Fixes sbase rm which
  compares stat("/") against target -- previously all inodes were 0,
  causing every file to match root. TTY fds get st_dev=2 with
  sequential inodes.
  Files: src/lib/posix_stat.c

## Priority 1: Signal Phase 4 -- EINTR on Interruptible Syscalls

read(), write(), nanosleep() should return -EINTR when a signal
is delivered during the call. Currently sig_check fires but the
syscall still returns its normal value.
Files: src/lib/posix_file.c, src/lib/posix_time.c

## Priority 2: Pipeline Cleanup on Mid-Fork Failure

If fork fails for segment N of M, earlier children are already running.
Fix: SIGKILL+reap already-forked children, close all pipe fds.
Files: src/apps/mini_shell.c

## Priority 3: getconf (last sbase tool)

Write scripts/gen_getconf.py to produce minimal getconf.h.
Would bring sbase to 99/99.
Files: scripts/gen_getconf.py

## Priority 4: Shared-memory pipes

Replace IPC round-trip with vka_alloc_frame mapped into both VSpaces.
Files: src/servers/pipe_server.c

## Priority 5: Allocator Memory Audit

The bump from 800 to 1500 pages is a band-aid. Audit untyped
consumption across boot, exec, fork, and thread creation to
right-size the pool. Consider whether vka_free_object properly
returns memory to the split allocator after CNode_Revoke, or
if a watermark approach is needed.

## Priority 6: PSCI Shutdown

Add PIPE_SHUTDOWN (label 77) IPC handler in aios_root.c that
issues PSCI SYSTEM_OFF (SMC 0x84000008) to cleanly power off
the QEMU virt machine. New aios_shutdown program sends the IPC.
Files: src/aios_root.c, src/apps/aios_shutdown.c
