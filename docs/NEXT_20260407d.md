## Priority 1: Multi-Chain Support (&&/|| recursion)

"true && echo a && echo b" drops the second chain. The chain
evaluation in mini_shell.c (~line 680) finds the first && or ||,
splits left/right, executes left, then processes right as a single
command. But right may itself contain && or ||.

Fix: iterative loop that re-scans right-side for further chain
operators, or recursive evaluation.

## Priority 2: POSIX Audit

* Create test program that reports implemented vs missing syscalls
* Test edge cases: dup2, symlink, chmod, chown, pipe()
* Fix access() for /etc/hostname (25/26 -> 26/26)

## Priority 3: Getty Robustness

* Fork failure retry with degraded fallback

## Priority 4: PIPE_EXEC_WAIT Deadlock Recovery

Child crash between PIPE_FORK and PIPE_EXEC leaves parent blocked.
handle_child_fault should call check_exec_wait(child_idx).
(Note: check_exec_wait is already called -- verify coverage.)

## Priority 5: Static Buffer Cleanup

do_exec/do_exec_bg use static buffers for argv. Move to stack-local.

## Priority 6: Signal Phase 3 -- Cross-Process Delivery

Current aios_sig_check() only runs on kill/tgkill return paths.
For cross-process signal delivery (parent sends SIGUSR1 to child),
the child needs to check sig_pending on other syscall return paths
(read, write, nanosleep). Add check points to blocking syscalls.
