## Starting from v0.4.72

## Completed in v0.4.72

* **tcc-compiled programs produce output.** Resolved the two
  blocking bugs preventing tcc binaries from running. tcc can now
  compile and execute C programs with full stdio, global variable
  access, and string literals on AIOS.
  Files: src/lib/aios_entry.c, tcc/arm64-gen.c

* **Shell > and >> redirect for builtins.** dash echo/printf can
  now redirect to files. dup2(file_fd, stdout) implemented via
  fd state copy so close() after redirect does not invalidate
  the target. Also fixed >> append mode (openat stores existing
  file size) and cat on 0-byte files (was returning ENOENT).
  Files: src/lib/posix_misc.c, posix_file.c, posix_internal.h, aios_posix.c

* **musl __sysinfo initialisation for tcc binaries.** tcc linker
  does not process .init_array constructors. The musl syscall
  dispatch pointer (__sysinfo) stayed NULL, causing every musl
  syscall (write, writev, etc.) to hit raw svc #0 and fault on
  seL4. Fix: __aios_entry sets __sysinfo = sel4_vsyscall and
  calls muslcsys_init_muslc() when the constructor was skipped.
  File: src/lib/aios_entry.c

* **tcc arm64 direct ADRP for static binaries.** tcc arm64_sym()
  used GOT-based addressing (R_AARCH64_ADR_GOT_PAGE) for all
  symbol references, even in static binaries where no GOT is
  loaded by the ELF loader. Every global variable or string
  literal access faulted. Fix: use direct PC-relative addressing
  (R_AARCH64_ADR_PREL_PG_HI21 + R_AARCH64_ADD_ABS_LO12_NC) when
  output_type is not TCC_OUTPUT_DYN.
  File: ~/Desktop/github_repos/tcc/arm64-gen.c

* **Empty file open fix.** openat returned ENOENT for 0-byte
  files because the read loop produced loaded=0 which triggered
  the error check. Fix: only error when fsize > 0 && loaded <= 0.
  File: src/lib/posix_file.c

## Bugs Resolved

### BUG 2 (from v0.4.71): tcc-compiled programs produce no output -- FIXED

Root cause was THREE separate issues, all required:
1. __sysinfo NULL (musl constructor skipped by tcc linker)
2. muslcsys_init_muslc not called (musl stdio not initialised)
3. arm64_sym uses GOT addressing in static binary (no GOT mapped)

Diagnostic methodology: seL4_DebugPutChar tracing (bypasses IPC),
systematic bisection of the call chain (TLS -> IPC -> syscall ->
stdio -> main), and targeted test programs (empty main, intra-module
call, global access, string literal) to isolate the failing
relocation type.

## Bugs Still Open

### BUG 1: virtio-blk device reads return stale data

**Severity: BLOCKING for persistent writes across cache eviction**

Status unchanged from v0.4.71. The 2048-entry ext2 block cache
mitigates the issue for same-session reads (tcc compile + exec
works because data stays cached). But after many I/O operations,
cache eviction causes subsequent reads of libc.a to fail
intermittently with "invalid object file" errors.

The bug does NOT block tcc compilation on a fresh boot. It does
block reliable repeated compilation and any use case requiring
persistent file writes readable after cache eviction.

Files: src/boot/blk_io.c, src/ext2.c

## Lessons Learned in v0.4.72

* **tcc .init_array constructors do not run.** tcc linker does
  not collect .init_array entries from archive objects or does
  not emit them in the output ELF. Critical for seL4 where
  musl and sel4runtime rely on constructors for init. Fix: call
  the constructors explicitly in __aios_entry.

* **tcc arm64 always uses GOT for symbol references.** arm64_sym()
  unconditionally emits R_AARCH64_ADR_GOT_PAGE + LD64_GOT_LO12_NC.
  Static binaries on seL4 have no GOT loaded by the ELF loader.
  Fix: emit ADR_PREL_PG_HI21 + ADD_ABS_LO12_NC for static output.

* **dup2(file_fd, stdout) then close(file_fd) invalidates redirect.**
  On a real OS, dup2 creates an independent fd copy. In AIOS, the
  redirect pointed to the original aios_fds entry which close()
  deactivated. Fix: snapshot the fd state into a static copy.

* **Bisection via return-value tests is powerful.** Testing
  return 42, return foo(), return x (global), return s[0] (string)
  in sequence isolated the exact relocation type (ADRP) that was
  broken, without needing to inspect binary internals.

* **seL4_DebugPutChar is the universal diagnostic.** It bypasses
  TLS, IPC, musl, and the tty server. Essential for debugging
  issues where any of those layers might be broken.

* **dash echo expands backslash-n.** echo 'hello\n' produces a
  literal newline, not the two characters \n. Use puts() or
  avoid \n in tcc test programs created via shell redirect.

## Priority 0: ext2 execute permission on created files

ext2_create_file() hardcodes inode.i_mode = 0x81A4 (-rw-r--r--).
Files written at runtime (tcc output, shell scripts) have no
execute bit. Need to either:
- Accept a mode parameter through the VFS create path
- Or set 0x81ED (-rwxr-xr-x) for all created files
- Or add chmod/fchmod support to set permissions after creation

Files: src/ext2.c (ext2_create_file), src/vfs.c (vfs_create),
       src/servers/fs_server.c (FS_WRITE_FILE handler)
