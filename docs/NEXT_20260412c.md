# AIOS Next Steps -- 2026-04-12c

## Completed in v0.4.86

### Ctrl-C signal delivery improvements

- **kill(0, sig) -> fg_pid in pipe_server**
  - POSIX `kill(0, sig)` now resolves pid 0 to `fg_pid` (foreground process)
  - SSH relay changed from `kill(child, 2)` to `kill(0, 2)` so Ctrl-C in SSH session targets the running child command (e.g., cat) instead of dash
  - Enables SSH Ctrl-C forwarding without process group support

- **Foreground parent promotion on child death**
  - When a foreground child is killed (e.g., SSH shell killed by Ctrl-C), `fg_pid` promotes to the parent process (e.g., sshd) instead of clearing to -1
  - Allows subsequent Ctrl-C to target sshd (first ignored via SIG_IGN, second force-kills)
  - Without this, killing an SSH child left fg_pid=-1 and sshd unkillable

- **Deferred net cleanup (seL4 reply cap fix)**
  - Root cause: `handle_child_fault` called `seL4_Call(net_ep_cap, ...)` to clean up sockets. In seL4 non-MCS, nested `seL4_Call` overwrites the reply cap needed to reply to the caller (root task Ctrl-C handler). Root task froze forever.
  - Fix: set `pending_net_cleanup_pid` flag in `handle_child_fault`, process the cleanup at the top of pipe_server main loop (after reply, before next `seL4_Recv`)

### TCP client-side connect()

- **Full 3-way handshake for outbound TCP connections**
  - `posix_net.c`: `connect(fd, addr, len)` sends NET_CONNECT IPC with socket_id, dst_ip, dst_port, caller_pid
  - `net_server.c`: NET_CONNECT handler resolves destination MAC via ARP (with gateway fallback), assigns ephemeral port, sends SYN, blocks caller via SaveCaller
  - TCP_SYN_SENT state added (net.h): connection lookup finds SYN_SENT sockets for SYN-ACK processing
  - SYN-ACK handler: completes 3-way handshake (sends ACK), transitions to TCP_ESTAB, wakes blocked connect()
  - RST handler: wakes blocked connect() with -ECONNREFUSED (111) on connection refused
  - SYN handler discriminates pure SYN from SYN-ACK (`!(flags & TCP_ACK)`) so SYN-ACK falls through to connection lookup

- **ARP cache exported**
  - `arp_cache_lookup()` made non-static in net_stack.c, declared in net.h
  - Used by NET_CONNECT for destination MAC resolution

### TCP protocol improvements

- **Dynamic window advertisement**
  - `tcp_tx_window` global (default 4096) replaces hardcoded 2048 in net_tcp.c
  - net_server sets window dynamically based on circular buffer free space before each ACK
  - Window accounts for incoming data size (`free_win = BUF_SZ - 1 - used - data_len`) to prevent over-advertisement

- **TCP sequence number validation**
  - `net_tcp_deliver()` rejects data where `seq != rcv_nxt` (retransmissions, out-of-order)
  - Sends duplicate ACK with current `rcv_nxt` for rejected segments
  - Prevents data duplication in circular buffer that caused SSH decryption failures

### Socket PID tracking and cleanup

- **Owner PID per socket**
  - `owner_pid` field added to `net_socket` struct
  - `socket()` resolves `aios_pid` via PIPE_GETPID (lazy init) before sending to net_server
  - net_server stores caller PID on socket creation (MR3 in NET_SOCKET)
  - connect() also sets owner_pid from caller

- **NET_CLEANUP_PID (label 98)**
  - New IPC handler in net_server: frees all sockets owned by a dying process
  - Sends TCP FIN for established connections
  - Deletes blocked recv, accept, and connect SaveCaller caps
  - Called from pipe_server `handle_child_fault` via deferred cleanup

- **sshd restart after SIGKILL fixed**
  - Root cause: killed sshd left listen socket + stale accept SaveCaller cap in net_server
  - New sshd SaveCaller failed (CNode slot non-empty), accept never returned
  - Fix: NET_CLEANUP_PID properly frees listen sockets, deletes stale accept/connect caps
  - PID resolution in socket() ensures cleanup matches the correct process

### Test program

- **tcp_connect**: client-side TCP connect test
  - Connects to specified IP:port (default 10.0.2.2:80)
  - Tests 3-way handshake, sends HTTP GET, reads response
  - Reports errno on failure (ECONNREFUSED = port closed, mechanism works)
  - Added to CMakeLists.txt as AiosPosixApp

## Known Issues

### SSH first connection attempt sometimes fails

- `enc-read: header failed` on first encrypted packet after auth probe
- Caused by TCP retransmission from SLIRP being rejected by sequence validation
- OpenSSH client retries automatically; second attempt always succeeds
- Proper fix: TCP retransmit timer + cumulative ACK (v0.4.87)

### SSH relay exit detection

- `ever_got_output` guard in channel_relay may delay EOF detection
- If child shell exits before producing output, relay ignores EOF
- Cosmetic: `exit` command works for normal sessions

### SSH Ctrl-C to child commands

- `kill(0, 2)` in SSH relay sends SIGINT to fg_pid via pipe_server
- Shell (dash) catches SIGINT but may not forward to running child
- Full fix requires process group tracking or PIPE_READ signal interruption
