  - `src/plat/plat.h` -- platform dispatcher (modeled after src/arch/arch.h)
  - `src/plat/blk_hal.h` -- block device HAL (matches ext2 blk_read_fn/blk_write_fn)
  - `src/plat/net_hal.h` -- network device HAL (TX + driver thread + MAC)
  - `src/plat/display_hal.h` -- display device HAL (framebuffer init + get)

- **Block I/O behind HAL**
  - `src/plat/qemu-virt/blk_virtio.c` -- virtio-blk sector read/write
  - Provides `plat_blk_read`, `plat_blk_write`, `plat_blk_read_log`, `plat_blk_write_log`
  - Identical logic to blk_io.c, renamed to HAL interface
  - boot_fs_init.c: `ext2_init(&ext2, plat_blk_read, 0)` (was `blk_read_sector`)
  - boot_log_init.c: `ext2_init(&ext2_log, plat_blk_read_log, 1)` (was `blk_read_sector_log`)

- **Build system changes**
  - CMakeLists.txt: `PLAT_QEMU_VIRT` compile definition
  - CMakeLists.txt: `src/` added to include path (for `plat/` headers)
  - CMakeLists.txt: `blk_io.c` replaced by `plat/qemu-virt/blk_virtio.c`

- **Cortex-A72 compatibility verified**
  - QEMU `-cpu cortex-a72` (RPi4 CPU): full boot to login prompt, all 4 cores
