- **src/plat/ directory structure**
  - `src/plat/plat.h` -- platform dispatcher (modeled after src/arch/arch.h)
  - `src/plat/blk_hal.h` -- block device HAL (matches ext2 blk_read_fn/blk_write_fn)
  - `src/plat/net_hal.h` -- network device HAL (TX + driver thread + MAC)
  - `src/plat/display_hal.h` -- display device HAL (framebuffer init + get)

- **Block subsystem behind HAL (Steps 1-3)**
  - `src/plat/qemu-virt/blk_virtio.c` (476 lines) -- virtio-blk init + sector I/O
  - Contains `plat_blk_init`, `plat_blk_init_log`, `plat_blk_read/write`, `plat_blk_read_log/write_log`
  - Virtio MMIO mapping, device probe, DMA allocation, legacy handshake -- all moved from boot_fs_init.c
  - Log drive init absorbed from boot_log_init.c (removed from build)
  - `boot_fs_init.c` reduced from 260 to 55 lines (pure VFS/ext2 mount)
  - Privatized: `blk_vio`, `blk_dma`, `blk_dma_pa` (+ log variants) as static in blk_virtio.c

- **Network subsystem behind HAL (Steps 4-5)**
  - `src/plat/qemu-virt/net_virtio.c` (330 lines) -- virtio-net init + TX + RX driver
  - Contains `plat_net_init`, `plat_net_tx`, `plat_net_driver_fn`, `plat_net_get_mac`
  - Virtio-net DMA, queue setup, feature negotiation, IRQ binding -- all moved from boot_net_init.c
  - RX driver thread moved from net_driver.c (removed from build)
  - TX function extracted from net_stack.c (net_tx_send is now a thin stats wrapper)
  - `boot_net_init.c` reduced from 195 to 15 lines
  - `net_stack.c` and `net_tcp.c` no longer include virtio.h
  - Privatized: `net_dma`, `net_dma_pa`, `net_irq_handler_cap` as static in net_virtio.c
  - Bridge globals remain: `net_vio`, `net_vio_slot` (set by blk_virtio, read by net_virtio)

- **Display subsystem behind HAL (Step 6)**
  - `src/plat/qemu-virt/display_ramfb.c` (212 lines) -- fw_cfg + ramfb framebuffer
  - Contains `plat_display_init`, `plat_display_get_fb`
  - fw_cfg MMIO mapping, file enumeration, ramfb DMA config, FB page allocation
  - `boot_display_init.c` reduced from 439 to 246 lines (font + rendering + boot text)
  - No fw_cfg/ramfb references remain in generic code
  - Privatized: `gpu_vio`, `gpu_dma`, `gpu_dma_pa`, `gpu_vio_slot` removed from root_shared.h

- **Build system**
  - `PLAT_QEMU_VIRT` compile definition added
  - `src/` added to include path for `plat/` headers
  - Source list: `blk_io.c` -> `blk_virtio.c`, `net_driver.c` -> `net_virtio.c`, added `display_ramfb.c`
  - `boot_log_init.c` removed from build (absorbed into blk_virtio.c)
  - `boot_services.c`: `net_driver_fn` -> `plat_net_driver_fn`

- **Cortex-A72 compatibility verified**
  - QEMU `-cpu cortex-a72` (RPi4 CPU): full boot to login prompt, all 4 cores
