## Completed in v0.4.88

### Infrastructure: morecore and allocator scaling

- **Morecore heap increased from 4MB to 8MB**
  - `LibSel4MuslcSysMorecoreBytes 8388608` in settings.cmake
  - Required for zsh runtime memory (zsh needs ~8-12MB at runtime)
  - 16MB was tested first but exhausted VKA page frames (4096 BSS pages per process)
  - 8MB is the safe upper limit with the current allocator pool

- **Allocator pool doubled from 4000 to 8000 pages**
  - `ALLOCATOR_STATIC_POOL_SIZE (BIT(seL4_PageBits) * 8000)` in aios_root.c
  - Provides headroom for the larger per-process BSS allocations
  - Full rebuild required (rm -rf build-04)

### ZSH Phase 1: script mode

- **zsh cross-compiled for AIOS (1.19MB AArch64 ELF)**
  - First alternative shell running on AIOS alongside dash
  - Script mode only (--disable-zle, no interactive line editing)
  - 7 static modules: main, rlimits, sched, datetime, langinfo, parameter, random
  - Disabled modules: zle, compctl, complete, complist, computil, zleparameter, termcap, ksh93, hlgroup, zutil

- **Build approach: configure + make + aios-cc re-link**
  - autoconf generates configure from configure.ac
  - configure runs with --host=aarch64-unknown-linux-gnu and libfakecurses.a stub
  - make compiles all .o files with cross-compiler (generates .mdh/.pro/.epro headers)
  - aios-cc re-links .o files with AIOS runtime (static, --wrap, seL4 libraries)
  - CFLAGS: -Wno-implicit-function-declaration -Wno-int-conversion (GCC 15 termcap compat)

- **Stubs required for zsh**
  - libfakecurses.a: termcap function stubs (tgetent, tgetstr, tgoto, tputs, setupterm)
  - Src/termcap.h: function declarations for the stubs
  - getrandom stub: reads /dev/urandom (not in seL4 musl)

- **Verified tests in AIOS**
  - `zsh -c "echo hello from zsh"` -> hello from zsh
  - `zsh -c "x=world; echo hello $x"` -> hello world
  - `zsh -c "for i in 1 2 3; do echo num: $i; done"` -> num: 1, 2, 3
  - `zsh -c "echo 2+3=$((2+3))"` -> 2+3=5
