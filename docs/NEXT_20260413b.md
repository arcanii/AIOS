### RPi4 boot chain learnings

- RPi4 EEPROM may request `start_cd.elf` instead of `start4.elf` (old EEPROM firmware)
- Fix: provide `start4.elf` renamed as `start_cd.elf` on FAT32 boot partition
- `gpu_mem=16` triggers cut-down firmware lookup (`start_cd.elf`); use `gpu_mem=64`
- seL4 elfloader for bcm2711 loads at `0x34f0000` (not default `0x80000`)
- `.start` section at file offset `0xf000`, entry point `0x34ff000`
- `.start` uses `adrp` (PC-relative) -- fully relocatable if relative layout preserved
- Bare-metal C: must use linker script + assembly entry stub to ensure `_start` is first
- VideoCore mailbox framebuffer: property channel 8, buffer address `& 0x3FFFFFFF` for ARM physical

## Next Priorities

### 1. RPi4 seL4 boot (requires serial adapter)

**Goal**: seL4 kernel + AIOS root task running on RPi4, serial output visible.

- USB-to-serial adapter: CP2102 or FTDI (~$5), GPIO 14 (TX) pin 8, GPIO 15 (RX) pin 10, GND pin 6, 115200 baud
- Boot the real seL4 image with trampoline:
  - `objcopy -O binary` the ELF to flat binary
  - Patch offset 0 with `b +0xf000` (ARM64: `0x14003C00`) to jump to `.start`
  - Set `kernel_address=0x34f0000` in config.txt
  - Or: use ELF directly (RPi firmware may parse ELF headers)
- Expected output: seL4 ELF-loader banner, kernel bootstrap, AIOS banner
- DTB will report: PL011 UART, eMMC, GENET, VC mailbox
- Block/net/display stubs will print "not yet implemented"
- Boot stops at getty spawn (no filesystem)

### 2. RPi4 Phase 2: SDHCI SD card driver

**Goal**: `ls /`, `cat /etc/passwd`, login prompt on RPi4 hardware.

- Implement `blk_emmc.c`:
  - BCM2835 SDHCI controller init from DTB (`hw_info.emmc_paddr`)
  - SD card init sequence: CMD0, CMD8, ACMD41, CMD2, CMD3, CMD7
  - PIO mode sector I/O: CMD17 (read), CMD24 (write)
  - IRQ binding for completion notification
- ext2 partition is partition 2 on the SD card
- `plat_blk_init()` returns 0, `plat_blk_read/write()` do sector I/O
- No log drive (single SD card -- `plat_blk_init_log()` returns -1)
- Reference: Linux `drivers/mmc/host/bcm2835-mmc.c`, Circle `lib/bcm2835-sdhci.cpp`
- Test: login on RPi4, `ls -la`, `cat /etc/passwd`, dash shell

### 3. RPi4 Phase 3: GENET Ethernet driver

**Goal**: SSH into RPi4, network commands.

- Implement `net_genet.c`:
  - BCM54213 GENET v5 MAC + MDIO PHY init from DTB (`hw_info.genet_paddr`)
  - DMA ring setup (256 TX + 256 RX descriptors)
  - RX poll into `net_rx_ring` (same SPSC interface as virtio-net)
  - TX via `plat_net_tx()` to descriptor ring
  - IRQ on RX completion -> `net_srv_ntfn_cap` signal
  - MAC address from DTB `local-mac-address` property
- Reference: Linux `drivers/net/ethernet/broadcom/genet/`, Circle `lib/bcmgenet.cpp`
- Test: `ping` from host, SSH session into RPi4

### 4. RPi4 Phase 4: VideoCore display integration

**Goal**: AIOS display server on HDMI, boot text, splash image.

- Implement `display_vc.c` (real driver, not bare-metal test):

  - `plat_display_get_fb()`: return mapped framebuffer pointer
- Wire into `boot_display_init.c` (gradient background, text rendering, splash)
