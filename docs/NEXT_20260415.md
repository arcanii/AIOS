## Features Implemented

### 1. Crypto Server (ChaCha20 CSPRNG)

Standalone server thread in aios_root providing /dev/urandom and getrandom()
via seL4 IPC. Replaces the weak splitmix64 PRNG.

**Architecture:**
- src/crypto/ (6 files): crypto_chacha20.c/.h, entropy_collect.c/.h, crypto_server.c/.h
- Runs as root task thread with its own endpoint (crypto_ep_cap)
- Entropy: ARM timer jitter (48-byte initial seed) + IPC timing deltas
- Auto-reseed every 4096 blocks (256 KiB output)
- Forward secrecy: reseed XORs into ChaCha20 key

**Integration points (CRITICAL for future cap additions):**
- boot_services.c: endpoint alloc + thread start
- exec_server.c: cap copy to children (EXEC_RUN/EXEC_RUN_BG path)
- pipe_server.c: cap copy to children (PIPE_EXEC fork+exec path) -- MUST UPDATE BOTH
- aios_posix.c: crypto_ep global, argv[7] parsing, strip 9 (was 8)
- posix_file.c: /dev/urandom read via crypto IPC (splitmix64 fallback)
- posix_compat.c: getrandom() via crypto IPC (splitmix64 fallback)

**Key lesson:** pipe_server.c has its OWN argv construction for PIPE_EXEC.
Any new cap must be added in BOTH exec_server.c AND pipe_server.c, along
with updating spawn_argv array size and bounds check.

### 2. aios-cc -c Detection Fix

**Bug:** `echo "$@" | grep -q "\-c"` matched `-c` as substring in flags
like `-Wno-sign-compare`, causing compile-only mode for link builds.
**Fix:** Changed to `echo " $* " | grep -q " -c "` (word boundary match).
This blocked TCC cross-compilation (built without -nostdlib, wrong CRT).

Installed to disk/rootfs/usr/ by mkdisk.py.

## Key Learnings

### Capability Passing (Two Code Paths!)

AIOS has TWO paths for spawning processes:
1. EXEC_RUN/EXEC_RUN_BG via exec_server.c (getty, initial process spawn)
2. PIPE_EXEC via pipe_server.c (fork+exec from shell)

Both maintain their OWN argv construction. When adding a new cap (like
crypto_ep), BOTH must be updated. The pipe_server also has spawn_argv
array bounds and MAX_USER_ARGV limits that must be adjusted.

### Stale Object Files

ninja may not detect source changes if the .obj timestamp is newer
than the source modification. When in doubt, delete the specific .obj:
```
rm build-04/projects/aios/CMakeFiles/aios_root.dir/__/__/src/servers/exec_server.c.obj
```
Then rebuild. A full `rm -rf build-04` is the nuclear option.

### aios-cc Fragility

The -c detection (`grep -q "\-c"` in args) matches substrings. Any flag
containing `-c` (like `-Wno-sign-compare`) triggers compile-only mode,
producing a default-linked binary (no -nostdlib, wrong CRT). Fixed to
word-boundary match.
